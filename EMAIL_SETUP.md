# Настройка отправки кодов на почту (Mail.ru)

Сайт отправляет коды подтверждения на email при регистрации, входе и смене почты. Чтобы письма реально уходили, нужно настроить SMTP. Ниже — пошаговая инструкция для **Mail.ru**.

---

## Где настраивать SMTP

В приложении **нет страницы настроек почты** в интерфейсе. SMTP задаётся **переменными окружения** двумя способами:

1. **Файл `.env` в корне проекта** (удобнее всего)  
   Скопируйте `.env.example` в `.env`, откройте `.env` и подставьте свой ящик Mail.ru и пароль приложения. При запуске приложение само подхватит переменные (нужен пакет `python-dotenv`: `pip install python-dotenv`).

2. **Переменные окружения в системе или в терминале**  
   Перед запуском задайте `MAIL_SERVER`, `MAIL_USERNAME`, `MAIL_PASSWORD` и т.д. (примеры ниже для Windows).

---

## 1. Параметры SMTP Mail.ru

| Параметр | Значение |
|----------|----------|
| Сервер   | `smtp.mail.ru` |
| Порт (TLS) | `587` |
| Порт (SSL) | `465` |
| Шифрование | TLS (рекомендуется) или SSL |

---

## 2. Включить доступ по паролю приложений в Mail.ru

1. Зайдите в [Почта Mail.ru](https://e.mail.ru).
2. Откройте **Настройки** (шестерёнка) → **Пароль и безопасность**.
3. Найдите блок **«Пароли для внешних приложений»** (или «Доступ по IMAP/SMTP»).
4. Включите **«Доступ по протоколу IMAP»** и **«Доступ по SMTP»**, если они выключены.
5. Создайте **пароль для внешнего приложения**:
   - Нажмите «Создать новый пароль» / «Пароль для приложения».
   - Укажите название (например, «Zazutu») и скопируйте выданный пароль.
   - Этот пароль используется только в переменной `MAIL_PASSWORD` (не ваш основной пароль от ящика).

Если такого раздела нет, в настройках аккаунта Mail.ru найдите «Пароль для приложения» или «Доступ по SMTP» и создайте пароль по инструкции на сайте.

---

## 3. Переменные окружения

Задайте переменные до запуска приложения.

**Через файл `.env`** (рекомендуется): в корне проекта создайте файл `.env` (можно скопировать из `.env.example`) и запишите в нём строки вида `MAIL_SERVER=smtp.mail.ru` (без пробелов вокруг `=`). Установите пакет: `pip install python-dotenv`.

**Через терминал** (только на время сессии):

### Вариант A: порт 587 (TLS)

```bash
set MAIL_SERVER=smtp.mail.ru
set MAIL_PORT=587
set MAIL_USE_TLS=1
set MAIL_USE_SSL=0
set MAIL_USERNAME=ваш_логин@mail.ru
set MAIL_PASSWORD=пароль_приложения_из_шага_2
set MAIL_DEFAULT_SENDER=ваш_логин@mail.ru
```

В PowerShell:

```powershell
$env:MAIL_SERVER="smtp.mail.ru"
$env:MAIL_PORT="587"
$env:MAIL_USE_TLS="1"
$env:MAIL_USE_SSL="0"
$env:MAIL_USERNAME="ваш_логин@mail.ru"
$env:MAIL_PASSWORD="пароль_приложения"
$env:MAIL_DEFAULT_SENDER="ваш_логин@mail.ru"
```

### Вариант B: порт 465 (SSL)

```bash
set MAIL_SERVER=smtp.mail.ru
set MAIL_PORT=465
set MAIL_USE_TLS=0
set MAIL_USE_SSL=1
set MAIL_USERNAME=ваш_логин@mail.ru
set MAIL_PASSWORD=пароль_приложения
set MAIL_DEFAULT_SENDER=ваш_логин@mail.ru
```

- `MAIL_USERNAME` и `MAIL_DEFAULT_SENDER` — полный email на Mail.ru (например `ivan@mail.ru`).
- `MAIL_PASSWORD` — **именно пароль для приложения** из раздела «Пароли для внешних приложений», не основной пароль от ящика.

## 4. Проверка

1. Запустите приложение.
2. Выполните регистрацию с реальным email на Mail.ru.
3. Проверьте входящие (и папку «Спам») — должно прийти письмо с кодом.

Если письма не приходят — чек-лист:

1. **Запустите приложение и посмотрите в консоль.**  
   - Должно быть: `[Почта] Настроена: сервер=smtp.mail.ru, отправитель=ваш@mail.ru`  
   - Если видите: `[Почта] НЕ настроена: письма будут выводиться в консоль` — переменные из `.env` не подхватились. Проверьте, что файл `.env` лежит в корне проекта (рядом с `app.py`) и установлен пакет: `pip install python-dotenv`.

2. **Проверьте содержимое `.env`:**  
   - Обязательно: `MAIL_USERNAME=ваш@mail.ru`, `MAIL_PASSWORD=пароль_приложения`, `MAIL_DEFAULT_SENDER=ваш@mail.ru` (или без `MAIL_DEFAULT_SENDER`, тогда подставится `MAIL_USERNAME`).  
   - Без пробелов вокруг `=`, без кавычек в значениях (или в кавычках, если так указано в примере).

3. **Пароль:** только пароль для внешнего приложения из настроек Mail.ru, не основной пароль от ящика.

4. **Порты:** для 465 — `MAIL_USE_SSL=1`, `MAIL_USE_TLS=0`; для 587 — `MAIL_USE_TLS=1`, `MAIL_USE_SSL=0`.

5. Если приложение пишет «Настроена», но письмо не пришло — при следующей попытке отправки (регистрация/вход/смена email) в консоли появится строка `[Почта] Ошибка отправки: ...` с текстом ошибки (например, неверный логин/пароль, таймаут). По ней можно понять причину.

---

## 5. Другие почтовые сервисы

- **Gmail**: `smtp.gmail.com`, порт 587, TLS; нужен «пароль приложения» в настройках аккаунта Google.
- **Яндекс**: `smtp.yandex.ru`, порт 587 (TLS) или 465 (SSL); пароль приложения в настройках Яндекса.

Логика та же: задайте `MAIL_SERVER`, `MAIL_PORT`, `MAIL_USERNAME`, `MAIL_PASSWORD`, `MAIL_DEFAULT_SENDER` и при необходимости `MAIL_USE_SSL`/`MAIL_USE_TLS` по документации сервиса.
